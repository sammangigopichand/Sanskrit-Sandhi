⚙️ Execution Workflow (The Pipeline)
When a user inputs a Sanskrit compound word (e.g., हिमालय), the system executes a 4-Layered Neuro-Symbolic Pipeline:

Layer 1: Lexicon Database Exact Match
The app queries the local SQLite database (

sanskrit_lexicon.db
).
If Found: It instantly returns the perfectly known historical split and rule ID. Execution stops here (Fastest & 100% Accuracy).
If Out-Of-Vocabulary (OOV): It passes the word down to the AI Layer.
Layer 2: PyTorch AI Inference (

backend/model/transformer.py
)
The word is converted into purely phonetic vectors (

backend/phonetics/features.py
), identifying grammatical traits (Place of Speech, Short/Long Vowels).
The PyTorch Multi-Task Transformer processes the word, simultaneously predicting 4 things:
Where the structural Sandhi split boundary should occur.
Which abstract Pāṇini mathematical Rule ID is occurring.
The reconstructed phonetic characters on the border sequence.
The Confidence Probability of its own prediction (e.g. 98% confident).
Layer 3: Hybrid Constrained Decoding (

backend/decoding/beam_search.py
)
Restricted Logit Masking: Before the model converts tensor values to output text, the Beam Search statically masks all non-Devanagari (Latin/English) characters to -inf. This guarantees no "garbage" generations.
Hard Constraints: The Rule Engine strictly scans the beam for invalid generations (e.g., continuous halants ्् or repetition loops ज्ञज्ञज्ञ) and kills invalid branches dynamically.
Soft Verification: The generated splits are fed into 

check_lexicon
 (

backend/symbolic/engine.py
). Valid words get a probability bonus, invalid words get penalized.
Layer 4: Symbolic Math Engine & Safe Fallback
If the AI Neural Network successfully navigates Layer 3 with high confidence, it outputs the AI-predicted string.
If the AI Confidence was low (< 60%) AND the Lexicon Soft-Scoring failed, the result routes directly to the Algorithmic Rule Engine (

backend/symbolic/engine.py
).
The Rule Engine attempts to mathematically un-merge Dirgha (ा, ी) and Guna (े, ो) markers manually.
Safe OOV Rejection: If the Algorithmic Rule Engine also cannot split the word, the system mathematically concludes that this string is not a valid Sandhi compound. It safely overrides and returns {"status": "unchanged"}, cleanly rendering as The Word is safely unsplit on the UI instead of printing hallucinations.
